# ОПИСАНИЕ ПРОГРАММЫ

## 1. Общие сведения

1.1 Программа представляет собой клиентское веб-приложение (SPA), реализованное на `React + TypeScript + Vite`.

1.2 Наименование программы: **Poprog Knowledge Base Frontend**.

1.3 Обозначение программы: **ИАЭ-ФРОНТ-КБ**.

1.4 Программа предназначена для работы в браузере и обеспечивает доступ к материалам портала (публикации, работы, проектные разделы, чат с ИИ-помощником).

1.5 Программа взаимодействует с backend API по HTTP/JSON; базовый адрес API задается через `VITE_API_BASE_URL` (при отсутствии используется локальная схема разработки с backend `http://localhost:8080`).

## 2. Функциональное назначение

2.1 Программа предназначена для:

- отображения контента базы знаний в веб-интерфейсе;
- навигации по разделам портала;
- выполнения поиска и перехода по найденным материалам;
- предоставления чата с ИИ-помощником.

2.2 Основные пользовательские функции:

- переход между страницами `/home`, `/publications`, `/works`, `/docs`, `/chat`, `/projects/:itemSlug`;
- открытие панели проектов и навигация по иерархии пунктов;
- ввод запроса в поиске, просмотр результатов, переход по `link` результата;
- предупреждение о переходе на внешний ресурс;
- работа со страницей `/not-found` при подтвержденном `404`;
- отправка сообщений в чат и загрузка истории по `chatId`.

## 3. Описание логической структуры

3.1 Программа построена по модульному принципу и включает:

- модуль маршрутизации (конфигурация роутов в `src/main.tsx`);
- модуль навигации и поиска (компонент `Navbar`);
- модуль страниц (`Home`, `PublicationsView`, `WorksView`, `DocsView`, `ChatView`, `ProjectItemView`, `NotFoundView`);
- модуль API-взаимодействия (`src/api/*`);
- модуль стилей (`src/styles/*`).

3.2 Логика поиска:

- ввод пользовательского запроса;
- обращение к endpoint `/api/search` с параметрами запроса;
- нормализация ответа и вывод карточек результатов;
- обработка перехода по `link` (absolute/relative URL, fallback-маршрут).

3.3 Логика перехода на ресурс из поиска:

- если ссылка внешняя и не относится к файлу на API-хосте, показывается модальное предупреждение;
- после подтверждения открывается новая вкладка с индикатором проверки;
- при подтвержденном `404` выполняется переход на `/not-found`;
- при невозможности достоверной проверки статуса (например, CORS) переход на ресурс не блокируется.

3.4 Логика чата с ИИ-помощником:

- отправка сообщения на `/api/assistant/chat`;
- чтение истории по `/api/assistant/chats/{chatId}/messages`;
- хранение `chatId` в `localStorage`.

3.5 Ограничения текущей реализации:

- маршрут `/projects` в текущей версии отображает контент главной страницы;
- неизвестные маршруты перенаправляются на внешний сайт лаборатории;
- корректность части внешних проверок зависит от политики удаленных ресурсов.

## 4. Используемые технические средства

4.1 Средства выполнения:

- персональный компьютер или мобильное устройство;
- современный веб-браузер (Chrome, Edge, Firefox, Safari).

4.2 Средства разработки и сборки:

- `Node.js` (рекомендуется не ниже `20.19`, либо `22.12+`);
- `npm`;
- `Vite`;
- `TypeScript`.

4.3 Сетевые средства:

- HTTP(S)-доступ к backend API;
- доступ к внешним ресурсам (для переходов из поисковой выдачи).

## 5. Вызов и загрузка

5.1 Подготовка окружения:

1. Установить `Node.js`.
2. Установить зависимости командой `npm install`.

5.2 Запуск в режиме разработки:

```bash
npm run dev
```

5.3 Сборка:

```bash
npm run build
```

5.4 Переменные окружения:

- `VITE_API_BASE_URL` — базовый адрес backend API.

Пример:

```bash
VITE_API_BASE_URL=http://localhost:8080
```

## 6. Входные данные

6.1 Входные данные от пользователя:

- клики по элементам интерфейса;
- поисковые запросы;
- текстовые сообщения чата;
- действия подтверждения/отмены в модальных окнах.

6.2 Входные данные от API:

- сгруппированные публикации (`/api/publications/grouped`);
- сгруппированные студенческие работы (`/api/student-works/grouped`);
- результаты поиска (`/api/search`);
- ответы чата (`/api/assistant/chat`);
- история сообщений (`/api/assistant/chats/{chatId}/messages`).

6.3 Формат данных:

- JSON (кодировка UTF-8);
- URL-параметры для поисковых запросов.

## 7. Выходные данные

7.1 Результаты работы программы для пользователя:

- отображение страниц и элементов интерфейса;
- визуальные состояния загрузки/ошибок;
- карточки поисковой выдачи;
- модальные предупреждения;
- страница `Ничего не найдено`;
- отображение истории и новых сообщений чата.

7.2 Результаты взаимодействия с внешней средой:

- HTTP-запросы к backend API;
- открытие внутренних и внешних ссылок в браузере;
- сохранение идентификатора `chatId` в localStorage.
