# Руководство пользователя

## 1. Назначение

Руководство описывает работу с веб-приложением базы знаний Poprog: навигацию, поиск, просмотр материалов и работу с чатом ИИ-ассистента.

## 2. Начало работы

### 2.1 Требования

- Современный браузер (Chrome, Edge, Firefox, Safari).
- Доступ к сети, где размещен frontend и backend API.

### 2.2 Открытие приложения

1. Откройте ссылку приложения в браузере.
2. После загрузки откроется главная страница (`/home`).

## 3. Основные элементы интерфейса

- Верхняя служебная панель (контакты, поддержка, аккаунт).
- Основное меню навигации (разделы портала).
- Кнопка поиска.
- Контентная область страницы.
- Нижний колонтитул (футер).

## 4. Навигация по разделам

Через основное меню доступны разделы:

- Главная (`/home`)
- Публикации (`/publications`)
- Дипломные работы и диссертации (`/works`)
- Документация (`/docs`)
- Чат с ИИ-помощником (`/chat`)

### 4.1 Раздел «Проекты»

1. Нажмите пункт «Проекты» в главном меню.
2. Откроется панель с группами и карточками.
3. В мобильном режиме сначала отображаются группы, затем элементы выбранной группы.

## 5. Поиск по порталу

### 5.1 Открытие поиска

1. Нажмите кнопку «Поиск» в шапке.
2. Введите текст запроса в строку поиска.
3. Дождитесь выдачи результатов.

### 5.2 Работа с результатами

Каждый результат содержит:

- тип материала,
- заголовок,
- дополнительное описание (авторы, дата и т.п.).

При клике по результату выполняется переход:

- если ссылка внутренняя, открывается соответствующий раздел портала;
- если ссылка внешняя, переход выполняется в новой вкладке;
- если ресурс не найден (подтвержденный `404`), открывается страница `Ничего не найдено`.

### 5.3 Предупреждение о внешнем ресурсе

Для внешних ресурсов перед переходом отображается предупреждающее окно.

1. Нажмите `Продолжить`, чтобы открыть ресурс.
2. Нажмите `Отмена`, чтобы остаться в приложении.

После подтверждения открывается новая вкладка с индикатором проверки ресурса, затем выполняется переход.

## 6. Страница «Ничего не найдено»

Страница открывается, когда система подтверждает, что целевой ресурс недоступен (`404`).

Действия пользователя:

- вернуться на главную страницу,
- повторить поиск с другим запросом,
- выбрать другой результат.

## 7. Чат с ИИ-помощником

### 7.1 Открытие чата

1. Перейдите в раздел `/chat`.
2. В верхней части отображается заголовок и описание режима чата.

### 7.2 Отправка сообщения

1. Введите текст в поле ввода.
2. Нажмите Enter для отправки.
3. Используйте Shift + Enter для переноса строки.

### 7.3 Особенности поля ввода

- Поле многострочное.
- Высота автоматически подстраивается по количеству строк.
- При превышении лимита строк включается внутренняя прокрутка.

### 7.4 История переписки

- История загружается автоматически, если в браузере сохранен `chatId`.
- При ошибке загрузки истории можно продолжить диалог в новом чате.

## 8. Адаптивные режимы

Приложение поддерживает:

- normal (полное меню),
- compact (сжатое меню),
- mobile (мобильное раскрывающееся меню).

Если часть элементов скрывается из шапки в компактных режимах, доступ к ним предоставляется через меню.

## 9. Типовые проблемы и решения

### Проблема: «Не удалось выполнить поиск»

Возможные причины:

- временная недоступность API,
- сетевой сбой.

Решение:

1. Проверьте соединение.
2. Обновите страницу.
3. Повторите запрос позже.

### Проблема: не открывается внешний ресурс

Возможные причины:

- ресурс удален или недоступен,
- внешний сайт ограничивает доступ,
- возвращается `404`.

Решение:

1. Повторите попытку.
2. Проверьте корректность ссылки в результате.
3. При необходимости используйте другой источник/материал.

## 10. Обратная связь

Для сообщений об ошибках и предложений используйте каналы связи, указанные в шапке/футере портала.

## 11. Ограничения текущей версии

- В разделе `Проекты` маршрут `/projects` в текущей версии ведет на главную страницу; детальные элементы открываются по `/projects/:itemSlug`.
- При переходе по внешним ссылкам проверка доступности зависит от ограничений браузера и удаленного сайта.
- Если внешний сайт ограничивает проверочные запросы, приложение открывает ресурс напрямую (без гарантии предварительной валидации статуса).
- История чата сохраняется только в текущем браузере через `chatId` в localStorage.
- При недоступности backend API поиск и чат могут работать с ошибками или быть временно недоступны.

## 12. Словарь терминов

- **API** — серверный интерфейс, через который приложение получает данные.
- **Внешний ресурс** — сайт/файл вне текущего веб-приложения.
- **Относительная ссылка** — путь без домена; система дополняет его базовым адресом API.
- **Абсолютная ссылка** — полный адрес ресурса (`http://...` или `https://...`).
- **Статус `404`** — ресурс по ссылке не найден.
- **Модалка** — всплывающее окно подтверждения действия.
- **Compact-режим** — промежуточный адаптивный режим интерфейса между desktop и mobile.
- **Mobile-режим** — режим для узких экранов с мобильным меню.
- **chatId** — идентификатор чата, позволяющий подгружать историю переписки.
