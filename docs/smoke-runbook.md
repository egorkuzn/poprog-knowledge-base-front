# Front Smoke Runbook

Документ для быстрой ручной проверки маршрутов, кнопок и служебных сценариев во фронтенде.

## Базовые маршруты

1. Открыть `/home` и проверить:
   - переходы в кейсы из карусели;
   - кнопки `Помочь проекту`, `Поддержка`, `Свяжитесь с нами`;
   - модальное окно после ответа на вопрос о полезности сайта.
2. Открыть `/publications`, `/works`, `/projects`, `/market`, `/docs`, `/video`:
   - хлебные крошки отображаются сразу под основным меню;
   - страницы не содержат placeholder-текста;
   - поиск/фильтры работают без ошибок в консоли.
3. Открыть `/projects/:itemSlug` для нескольких slug:
   - `reflex`
   - `ride-overview`
   - `edtl-spec`
   - `quality-metrics`

## Личный кабинет

1. Перейти на `/account?mode=login`.
2. Выполнить вход через внутреннюю форму.
3. Проверить:
   - загрузку профиля, донатов, избранного и чатов;
   - экспорт истории в `CSV` и `PDF`;
   - действие `Повторить донат`.

## Служебный сценарий

1. На `localhost` выбрать роль `ADMIN` или `SUPPORT` во внутренней форме входа.
2. После входа перейти в `/account/admin/donations`.
3. Проверить:
   - загрузку KPI по донатам;
   - фильтры по статусу и диапазону дат;
   - пагинацию таблицы;
   - административный экспорт `CSV/PDF`.

## Ошибки и деградации

Если любой маршрут показывает `Failed to fetch`, проверить:

1. значение `VITE_API_BASE_URL`;
2. доступность backend на указанном порту;
3. активность local/dev header auth в backend;
4. наличие локальной сессии в `localStorage`.
